# Primo — Internal Tools

Outils internes pour l'équipe Primo, hébergés sur GitHub Pages.

## Fichiers

| Fichier | Description | URL GitHub Pages |
|---|---|---|
| `brief-avant-vente.html` | Brief avant-vente AE → SE | `https://<username>.github.io/<repo>/brief-avant-vente.html` |

## Déploiement

1. Créer un repo GitHub (ex: `primo-tools` ou `primo-internal`)
2. Pousser ce dossier : `git init && git add . && git commit -m "init" && git remote add origin <url> && git push -u origin main`
3. Aller dans **Settings → Pages → Branch: main / root → Save**
4. L'URL sera disponible dans **Settings → Pages** (~2 min)

## Embed dans Notion

1. Dans la page Notion cible, taper `/embed`
2. Coller l'URL GitHub Pages du fichier
3. Ajuster la hauteur du bloc embed (~900px recommandé)

## Usage

Les AEs remplissent le formulaire directement depuis Notion.
Cliquer sur **"Générer & copier le brief"** — un résumé formaté est copié dans le presse-papier, prêt à être collé dans Slack ou envoyé par email au SE.

> Raccourci clavier : `Cmd+Enter` (Mac) / `Ctrl+Enter` (Windows) pour générer le brief.
